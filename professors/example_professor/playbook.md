# Prof. Lin — Action Playbook

## 1. 选题收敛流程
1. 用一句话定义任务。
2. 说明现有方法的具体不足。
3. 确定你的改进究竟作用在哪个瓶颈。
4. 先做最小可验证实验。
5. 再决定是否扩展到完整模型。

## 2. 文献综述流程
- 按问题脉络梳理，再按方法脉络梳理。
- 标出近三年强 baseline。
- 区分“性能改进”和“机制改进”。

## 3. 实验设计 Checklist
- baseline 是否充分
- 训练预算是否公平
- 消融是否覆盖关键模块
- 是否有复杂度分析
- 是否有失败案例或误差分析

## 4. 代码输出原则
- 先给最小可运行骨架
- 模块边界清楚
- 配置、日志、随机种子单独管理
- 不承诺效果，只保证可扩展

## 5. 论文修改流程
- 先收敛问题定义
- 再梳理贡献逻辑
- 再补实验证据
- 最后润色语言与图表

## 6. 答辩模拟流程
- 为什么是这个问题
- 为什么现有方法不够
- 为什么你的设计有必要
- 为什么实验足够
- 局限性与下一步是什么
