# Research Memory 分析 Prompt

## 任务

你将收到关于某位教授的论文、主页、讲义、报告、访谈或人工总结。
请从中抽取该教授可复用的 **研究知识结构**，用于构建 `research_memory.md`。

## 提取维度

### 1. 研究身份
- 所属学科 / 子方向
- 典型问题域
- 研究对象、任务、系统或场景
- 常见术语与表述方式

### 2. 核心问题清单
- 该教授反复研究的关键科学问题 / 工程问题
- 这些问题为什么重要
- 其典型问题定义方式

### 3. 方法谱系
- 常用方法类型
- 方法随时间的演化
- 反复出现的结构、模块、约束、假设
- 偏爱的实验范式和评价指标

### 4. 代表成果
- 代表论文 / 代表项目
- 各自的核心贡献
- 常被用来证明其研究风格的工作

### 5. 判断标准
- 他/她通常把什么视为“创新”
- 什么只是工程堆叠，什么算机制性贡献
- 对实验完备性、公平性、消融、可解释性的重视程度

### 6. 边界与禁区
- 哪些结论容易被过度泛化
- 哪些假设在原材料中被反复强调
- 哪些场景缺乏证据，不应擅自延伸

## 输出要求

- 只写有依据的内容。
- 材料不足时明确标注“原材料不足”。
- 尽量使用学术化、结构化表述。
- 给出“来自材料”的结论，不要把常识误包装成教授独有观点。
