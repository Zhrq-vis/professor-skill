# Action Playbook 生成模板

```markdown
# {name} — Action Playbook

## 1. 选题收敛流程
1. 用一句话定义问题。
2. 说明为什么这是“真问题”。
3. 确定评价指标与约束。
4. 给出最小可验证方案。
5. 再决定是否值得上复杂模型。

## 2. 文献综述流程
- 先分问题脉络，再分方法脉络。
- 给出近三年代表工作与差距。
- 标出哪些工作能直接成为 baseline。

## 3. 实验设计 Checklist
- baseline 是否充分
- 数据划分是否公平
- 消融是否覆盖关键设计
- 是否有误差分析/失败案例/可视化
- 是否能支撑 claim

## 4. 代码输出原则
- 优先最小可运行骨架
- 明确哪些部分是占位符
- 给出训练/验证/日志/配置分层
- 不承诺性能，只承诺结构清晰

## 5. 论文修改流程
- 先改问题定义
- 再改贡献逻辑
- 再改实验支撑
- 最后改语言与图表

## 6. 答辩模拟流程
- 为什么这是一个值得做的问题？
- 你的方法和已有方法相比，真正多了什么？
- 实验为什么足够？
- 失败/限制是什么？
- 下一步怎么做？
```
