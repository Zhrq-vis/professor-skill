# 赛博教授基础录入脚本

## 开场白

```text
我来帮你创建一个“赛博教授” Skill。

这个过程不是简单角色扮演，
而是把一位教授的研究轨迹、指导风格和方法论，蒸馏成一个能带学生做科研的智能体。

我会先问你几个基础问题，再接收论文、主页、讲义、访谈或你的人工总结。
```

## 问题序列

### Q1：教授代号 / 姓名简称（必填）

```text
先给这位教授一个代号吧。
可以是真名简称、英文缩写，或者一个方便调用的名字。

比如：chen-lab / prof-wang / cyber-sci-advisor
```

### Q2：学科与研究方向（可跳过）

```text
用一句话介绍这位教授所在的学科、主要方向、院校/实验室。

示例：
  “计算机视觉，医学图像分析，某高校智能医学实验室”
  “量子计算，量子控制与优化，某研究院”
```

### Q3：你想让这个赛博教授主要承担什么角色（可多选）

```text
例如：
- 导师型：帮学生收敛问题、设计实验
- 审稿人型：专门挑逻辑和实验问题
- 写作教练型：按该教授风格修改论文
- 代码教练型：输出最小可运行代码骨架
- 答辩官型：模拟组会/开题/预答辩追问
```

### Q4：你准备提供哪些材料

```text
可以混合提供：
- 教授论文 PDF
- 主页 / Scholar / DBLP / 实验室介绍
- 课程讲义 / keynote / 访谈
- 你自己写的一份“教授画像”
- 学生当前草稿 / 代码 / 实验计划
```

## 汇总确认

```text
好的，我整理一下：

  教授代号：{name}
  学科方向：{summary}
  目标角色：{roles}
  输入材料：{sources}

确认后我开始蒸馏。
```
